## Versioning

MCP Studio follows [Semantic Versioning](https://semver.org/):
- **MAJOR** version for incompatible API changes
- **MINOR** version for added functionality in a backward-compatible manner
- **PATCH** version for backward-compatible bug fixes

## Release Schedule

We aim for regular releases following this schedule:
- **Patch releases**: As needed for critical bug fixes
- **Minor releases**: Every 4-6 weeks with new features
- **Major releases**: Every 6-12 months with significant changes
